\doxysubsection*{Adding a new problem to Dice QMC}

There are two main components to {\itshape \mbox{\hyperlink{namespace_dice_py}{Dice\+Py}}}\+: the Input Generator and the Visualisation library. We will handle these sections separately.

\doxysubsubsection*{Input Generator}

The Input Generator {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__inputs}{Dice\+Py.\+dice\+\_\+inputs}}} incorporates three other modules\+:


\begin{DoxyItemize}
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__defaults}{Dice\+Py.\+dice\+\_\+defaults}}} lists the default variables needed throughout the Input Generator, made to match the defaults in the Fortran backend code.
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__check}{Dice\+Py.\+dice\+\_\+check}}} is used to check existing {\itshape params.\+txt} files for any errors, to prevent further errors when running the Fortran backend code.
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__write}{Dice\+Py.\+dice\+\_\+write}}} is used to write new {\itshape params.\+txt} files, which are made by the Input Generator
\end{DoxyItemize}

\doxyparagraph*{{\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__defaults}{Dice\+Py.\+dice\+\_\+defaults}}}}

Implementation of a new problem into {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__defaults}{Dice\+Py.\+dice\+\_\+defaults}}} is perhaps the simplest of the three. Aside from implementing the new system into the {\ttfamily systems} list at the top of the module, the user will need to extend the lists for the globally shared parameters such as {\ttfamily d\+\_\+vqmc\+\_\+steps} and {\ttfamily d\+\_\+sigma}, and add in new variables for problem-\/specific parameters such as in the {\ttfamily Parameters for QHO} and the {\ttfamily Parameters for H2plus} sections.

The user will also need to add in a new {\ttfamily porder} list. These lists specify the order that parameter tags should be iterated through when checking an input file, in order to be consistent with files made by the Input Generator.

\doxyparagraph*{{\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__check}{Dice\+Py.\+dice\+\_\+check}}}}

{\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__check}{Dice\+Py.\+dice\+\_\+check}}} makes use of a number of inner checking routines to ensure that values in a user\textquotesingle{}s {\itshape params.\+txt} are valid. When adding a new problem these do not need to be changed, but they will still be utilised within the {\ttfamily check\+\_\+inputs} function. Most of this function can remain the same, but there are a couple of problem-\/specific sections that will need to be added to.

The first section starts from the comment {\ttfamily \# Check p\+\_\+system of PARAMETERS namelist}. This section checks for the presence of the PARAMETERS namelist, and populates it if it is found to be empty. A {\ttfamily elif p\+\_\+system==\char`\"{}your\+\_\+system\char`\"{}\+:} section will need to be added to the end here that automatically populates the PARAMETERS namelist with the values you just defined in {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__defaults}{Dice\+Py.\+dice\+\_\+defaults}}}.

The second section also requires a similar extension to an existing {\ttfamily elif} statement, which checks through the system-\/specific variables supplied in the user\textquotesingle{}s {\itshape params.\+txt} and determines if they are of the correct type (boolean, float, etc.) using the aforementioned inner checking routines. Of note here are\+:


\begin{DoxyItemize}
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__check_afa56e066426982c324e32bd0ad8ad2bf}{isvalid()}}}, which checks if a given {\ttfamily tag}\textquotesingle{}s {\ttfamily values} match one of the expected {\ttfamily options} from a predefined list.
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__check_a98dca2498872f2c2a52c4094512f1a9b}{check\+\_\+float()}}}, which checks if a given {\ttfamily tag}\textquotesingle{}s value(s) are floats, and replaces them with the {\ttfamily tag}\textquotesingle{}s default values if not.
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__check_a98e55f88fd3e099f39b2b2b773b0e720}{check\+\_\+grid()}}}, which does the same as above but for a grid-\/type parameter, ie. a {\ttfamily (start, stop, num)} set of numbers to run a linear spacing function on, similar to numpy\textquotesingle{}s {\ttfamily linspace}.
\end{DoxyItemize}

\doxyparagraph*{{\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__write}{Dice\+Py.\+dice\+\_\+write}}}}

{\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__write}{Dice\+Py.\+dice\+\_\+write}}} makes use of a similar set of checking functions to the above, but also contains a set of querying functions. These prompt the user for a response in a variety of places throughout the Input Generator, and are made to be extensible to new problems being added. Of note here are\+:


\begin{DoxyItemize}
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__write_a876080e22b8a59221886abb1d420176d}{isint()}}} and {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__write_a574df547f033dc2f9681db9085915822}{isfloat()}}}, which check if the string {\ttfamily s} that they are passed is an integer or a float, respectively. These are used consistently throughout the writer to ensure that the user is providing the correct kind of input and not, for example, passing a float to the variable for the total number of VQMC steps, which should always be an integer.
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__write_abd52e4ec35811b04e3525b0f3db3a74f}{query\+\_\+default()}}}, which queries if a user wishes to use the {\ttfamily default} value for a {\ttfamily param}, as described in {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__defaults}{Dice\+Py.\+dice\+\_\+defaults}}}.
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__write_a72a80ad03ea22ae6afae9e9cb7e97a57}{query\+\_\+step()}}}, which asks the user for a positive integer ie. a \textquotesingle{}step\textquotesingle{} value, and {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__write_a511124193533a3328fbd919cb9eca1e4}{query\+\_\+posfloat()}}}, which does the same for positive floats.
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__write_ad3bd71b7240382fbbdb2a0eb04f129cb}{query\+\_\+grid()}}}, which constructs a {\ttfamily (start, stop, num)} grid to be linearly spaced once in the Fortran backend, while checking that values in the grid are of the correct type to avoid errors.
\end{DoxyItemize}

The majority of the module is located in the {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__write_a9c61faf451621bddc76ed8a230036039}{query\+\_\+params()}}} function, which runs the user through the entirety of input generation using the above routines. Much of this function is not specific to a new problem and can be left as-\/is, but there are sections that check the value of the {\ttfamily sys} variable that will need to be constructed for a new problem. \doxyparagraph*{{\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__inputs}{Dice\+Py.\+dice\+\_\+inputs}}}}

This module just contains the {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__inputs_a261569dce72b6b21ec3f73f090fffb4a}{dice\+\_\+inputs()}}} function, which is simply a caller for the other modules. No extra work should be needed here.

\doxysubsubsection*{Visualisation}

\mbox{\hyperlink{namespace_dice_py}{Dice\+Py}} reads in results using the {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__visualise}{Dice\+Py.\+dice\+\_\+visualise}}} module, and is able to visualise the results generated by the main program. Writing additional visualisation options for different problems can be by simply adding a function to plot the positions sampled by the Markov chain, and a function which plots the energies and their uncertainties against the variational parameter.

For example, for the QHO problem\+:
\begin{DoxyItemize}
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__visualise_a592dd6c19dea285182a2d766eba39393}{QHOEnergy\+Plot()}}} reads {\ttfamily results\+\_\+file} and plots the energies against variational alpha values specified in the problem, as well as the uncertainties in the energies. The sliceby option allows one to plot only a section of the graph by typing the indices of the slice you are interested in.
\item {\ttfamily \mbox{\hyperlink{namespace_dice_py_1_1dice__visualise_a9b1db186f691296871f6344f4bff4e67}{QHOWfn\+Plot()}}} reads a file containing the Markov chains and plots the distribution of the QHO wave functions, also showing the energies of these positions as well. It can also plots an analytical solution for the wave function as well, if desired.
\end{DoxyItemize}

Plots may want to take advantage of the {\ttfamily plotly} or {\ttfamily matplotlib} libraries which have been used for the other functions in the module, but users can also decide to use other libraries as well when writing these functions.
